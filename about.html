Pebble Match

Pebble Match is a portfolio project we made at Holberton School. It’s an end of year project, part of your first year. We decided to make a web application called “Pebble Match”. What is “Pebble Match”? And who are we?

What is “Pebble Match”?

Pebble Match is a matchmaker website. There is no judgment on people’s looks. Pebble Match matches people based on questionnaire on habits and values, which is a fundamental (albeit often overlooked) element in a relationship.

Who are we?

We are students of Holberton School in Paris, a Software Engineering school. We have been in this school since 16 May 2022. We call our team “The Super Team”. Here are the members of “The Super Team” and their roles:

Whom did we create this website for?

This website is for single people (from 18 years old) looking for a relationship. A user can find their partner by answering questions related to their daily life, so they can find their partner based on shared interests.

What did we focus on?

For this project, features related to the user are the most important. We set up a database to store information on users. Before matching a user with another one, we need to save users’ answers to our questionnaire in our database. After that, we try to match users based on common answers.

Why we choose this project?

One day, when Mia and Noémie were trying to find an idea for their portfolio project, they talked about Chong and how they were married whereas Chong was single. So they decided to make this project to find a girlfriend for Chong, and invited Chong to join this project. That’s a little story about the project “Pebble Match”.

What’s the link between Pebble Match related and penguins?

Have you noticed that there are some little penguins waddling around on the site?
Yes! They have a story too!
Among some species of penguins (Adelie, Chinstrap and Gentoo), a male penguin searches for the prettier pebble to propose to the penguin of its life. (After all, humans also propose with a diamond). If the other penguin likes it, it will bring it back to its nest, then the two love birds can build up their pebble nest to raise their family!

Technologies that we use in this project

For the front-end, we used HTML, CSS, JavaScript and the framework Bootstrap.

For the back-end, we used the Python language with the framework Django.

For the data structure, we used an open-source relational database management system called MySQL. It’s so complicated to use, but we try to use it to gain experience.

Features

You can register as a new user and start answering the questionnaire.

After you’ve answered questions, you can see the top three people whom you’re supposed to match with. And then you can send a pebble to another user that appears on the match. If another user accepts your pebble, this means they want to start a relationship with you. If not, send it to another user.

The website is responsive and mobile compatible. This means users can access the site with a mobile view.

Most difficult technical challenge

For the back-end, the most difficult challenge was the connection between framework, code and database. MySQL was the biggest challenge for us. When we started our project, we lost a lot of time trying to solve problems related to the database. At first, none of our user data was saved in the database. But the team didn’t give up, we persevered, and finally we succeeded.

For the front-end, the most difficult challenge was JavaScript. During our previous quarter at Holberton School, we had only 3 JavaScript projects, so we didn’t know all the syntax rules about this language. But we also managed it, we searched for resources on the Internet, and learned to use it. We finally made it.

Technical take-aways

Front-end and database are not as simple as we thought. It’s not just CSS, HTML, Excel, Data and design. New technologies like JavaScript, Bootstrap and MySQL make it more complex and a lot of logic should be used for the front-end and database.

We have seen the whole stack so as a future engineer it is a good overview of what we could expect in a professional environment.

We also learned that it is not always easy to work in a group and that communication is essential to ensure the success of the project until the end. Communication is the key in big projects. We also made extensive use of Git branches, which saves a lot of time and is a precious tool when working in a team.
